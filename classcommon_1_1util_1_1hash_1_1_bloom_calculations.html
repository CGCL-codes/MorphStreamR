Detailed Description

The following calculations are taken from: http://www.cs.wisc.edu/~cao/papers/summary-cache/node8.html "Bloom Filters - the math"

This class's static methods are meant to facilitate the use of the Bloom Filter class by helping to choose correct values of 'buckets per element' and 'number of hash functions, k'.

static BloomSpecification common.util.hash.BloomCalculations.computeBucketsAndK ( double  maxFalsePosProb)
static

Given a maximum tolerable false positive probability, compute a Bloom specification which will give less than the specified false positive rate, but minimize the number of buckets per element and the number of hash functions used.

Because bandwidth (and therefore total bitvector size) is considered more expensive than computing power, preference is given to minimizing buckets per element rather than number of hash funtions.

Parameters
maxFalsePosProbThe maximum tolerable false positive rate.
Returns
A Bloom Specification which would result in a false positive rate less than specified by the function call.
